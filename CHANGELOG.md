# CHANGELOG

## [0.38.0] - $(date +%Y-%m-%d)
- Automated synchronization from upstream.
- Conducted intelligent merge resolution on active feature branches.

$(cat CHANGELOG.md | grep -v "# CHANGELOG")
- Implemented Python language bindings using `pyo3` allowing developers to execute Python logic scripts that interface with the `NativeUI` Rust bridge.
- Refined the Event Bridging layer by implementing precise tree-based `hit_test` logic to map cursor and click events strictly to target nodes rather than generic window coordinates.
- Enhanced Autonomous Task Scheduler: The JavaScript scheduler now dynamically triggers a full `RunPipeline` recovery mechanism if performance metrics stall (FPS < 5) to ensure runtime stability and self-healing.
- **Watchdog Execution Protocol:** Implemented the `RunPipeline` UI Command to execute the Node.js End-to-End (`test:e2e`) pipeline externally. This enables the engine to completely test and recover itself when the JavaScript scheduler detects fatal performance degradation.
- **System-Aware Resource Orchestration:** Upgraded the `src/monitor.rs` loop to actively monitor the host machine's total RAM usage alongside the application's RSS memory fingerprint. When memory pressure exceeds 85%, the engine aggressively scales down its internal cache structures (Text, Textures) and shrinks render batch sizes to prevent OOM termination.
- **Core System Integration & Lifecycle Validation:** Implemented an automated E2E lifecycle verification sequence during engine boot in `src/runtime.js`.
- **Telemetry Dashboard Integration**: `AppStats` generated by the core Rust execution loop are now explicitly propagated back across the QuickJS boundary via `globalThis._latest_stats`. This allows the JS runtime (and any AI-generated UI scripts) to read real-time layout latencies, batch sizes, and resource consumption.
- **E2E Lifecycle Validation**: Established a comprehensive `runE2eLifecycleValidation` check on boot that explicitly tests native engine responsiveness, metrics readback, and core bridge operations, escalating immediately to watchdog recovery upon failure.
- **Autonomous Task Execution & Runtime Automation**: Integrated a background task scheduler logic into `src/runtime.js`. The engine now utilizes telemetry data (FPS, CPU idle capacity) to intelligently spawn background background operations (e.g. AI-driven compilation scripts `UiCommand::RunAutonomousTask`) entirely decoupled from the main UI thread.
- **Transpiler E2E Verification:** Manually executed and verified the `ui_gen.rs` compilation pipeline using mock JSON structural ASTs. The pipeline correctly handles code generation and successfully passes the `cargo check` phase.
- **Autonomous Task Validation:** Implemented an explicit execution check in `src/runtime.js` that tests deep native UI bridge sanity (node creation, metrics fetching) immediately on engine boot to validate transpiler loops without requiring `GEMINI_API_KEY` dependencies.
- **SVG / Vector Graphics Pipeline:** Finalized the SVG rendering pipeline integration. Enhanced `render_svg_to_rgba` within `src/render.rs` using `usvg` and `tiny_skia` to actively scale SVG structures mapped via Taffy Layout constraints (`width/height`) into dynamic GPU texture cache buffers.
- **Documentation**: Overhauled `README.md` to remove the outdated "Under Construction" Jules prompt template, replacing it with the formal v0.38.0 engine specification, architecture overview, and updated `npm run test:e2e` quickstart commands.
- **Visual Regression Suite:** Finalized the automated visual regression script (`scripts/visual_test.js`) to explicitly check for the native WGPU `capture_frame` pipeline and map any dumped screenshot artifacts for automated pixel-diffing in future E2E runs.
